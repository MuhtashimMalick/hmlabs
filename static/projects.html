AttackEye
EASM Platform

Type
Cybersecurity / SaaS
Origin
NED University — NCCS
Status
Live Product
Tech Stack
Amass Nmap Sigma.js Python FastAPI
attackeye.ai →

The Problem

Organizations struggle to maintain visibility over their own external attack surface. Subdomains, exposed ports, third-party assets, and dark-web mentions accumulate faster than security teams can track them manually. AttackEye was built to make external attack surface management continuous and automated.

What We Built

Built an external attack surface management (EASM) platform that continuously maps and monitors an organization's internet-facing footprint.
Integrated automated domain enumeration using Amass to discover subdomains, related assets, and shadow IT the organization may not be tracking.
Wired in port and vulnerability scanning via Nmap to surface exposed services and misconfigurations across discovered assets.
Added dark-web monitoring and brand protection capabilities to detect credential leaks, brand impersonation, and threat actor mentions.
Visualized the entire attack graph using Sigma.js network graphs, giving security teams an interactive, explorable view of their exposure.
Outcome

AttackEye is a live product born from research at NED University's National Centre for Cyber Security. It gives security teams continuous, automated visibility over their external exposure.

The Problem

Property managers handling rental and sale listings across multiple properties had no unified system for reservations, smart device control, and tenant management. Existing tools handled one piece of the puzzle but nothing connected the physical layer (IoT devices) with the business layer (bookings, listings, operations).

What We Built

Built a full-stack IoT property management platform that unifies listings, reservations, and connected-device control in a single system.
Engineered real-time device management — property managers can monitor and control smart devices across properties from a central dashboard.
Built a reservation and listing engine that handles availability, booking flows, and property state transitions end-to-end.
Designed the backend in FastAPI with a React frontend, optimized for low-latency device command delivery and real-time status updates.
Architected the system to support both rental and sale property workflows with configurable listing types and device profiles per property.
Outcome

EzeHome is a live product connecting property operations and smart devices in one platform, available at ezehome.net.

Project 03

MorfBot
Voice AI Agent

Type
Voice AI / Agentic System
Status
In Production
Tech Stack
Twilio Deepgram STT ElevenLabs TTS LLM Reasoning FastAPI

The Problem

Sales teams spend a disproportionate amount of time on outbound calls — qualifying leads, handling objections, and booking follow-ups — work that doesn't require a human but historically had no good automated alternative. Existing voice bots were stiff, slow to respond, and broke down the moment a conversation deviated from a script.

What We Built

Built a real-time voice AI sales agent using Twilio Conversation Relay as the telephony layer — handles inbound and outbound calls end-to-end.
Integrated Deepgram for low-latency speech-to-text transcription, tuned for natural conversational cadence and interruption handling.
Wired in ElevenLabs for natural-sounding voice synthesis, giving the agent a consistent, human-like persona across all calls.
Built an LLM reasoning loop that drives context-aware responses, handles objections, and steers conversations toward qualification or booking goals.
Designed the entire pipeline for sub-second end-to-end latency — the gap between a caller finishing a sentence and MorfBot responding is imperceptible.
Outcome

MorfBot is live in production, autonomously handling sales calls with natural conversation flow — demonstrating that voice AI can move beyond scripted bots into genuine agentic reasoning.

Project 04

SOS ↔ StockTrim
Sync Engine

Client
US-Based Supply Chain Co.
Type
Integration Engineering
Status
Live in Production
Tech Stack
FastAPI REST APIs PostgreSQL Background Workers Docker

The Problem

A US-based supply chain company was running two independent enterprise platforms — SOS Inventory and StockTrim — with no automated way to keep them in sync. Inventory data had to be reconciled manually, introducing lag, errors, and operational overhead that scaled badly with their catalog size.

What We Built

Designed and built a secure two-way data synchronization engine between SOS Inventory and StockTrim, handling full bidirectional state propagation.
Built a scheduled automation layer with configurable sync intervals — zero manual intervention required after deployment.
Implemented intelligent schema mapping and transformation logic to bridge mismatched data models across both platforms without data loss.
Engineered robust handling for API rate limits, partial failures, and retry recovery — the system degrades gracefully and self-heals.
Applied idempotent processing and conflict resolution strategies to prevent duplicate or inconsistent records across sync cycles.
Added structured end-to-end logging for full traceability of every sync operation, surfaced through a frontend log viewer.
Outcome

The system eliminated manual inventory reconciliation entirely. The client now has real-time, consistent inventory visibility across both platforms with no operational overhead.

Unleashed ERP
Migration

Client
Australian E-Commerce & Distribution Co.
Type
ERP Migration & Backend Engineering
Status

Tech Stack
Python Unleashed API MYOB Data Pipelines Backend Automation

The Problem

An Australian e-commerce and distribution company needed to migrate away from MYOB — a legacy accounting and inventory system — to Unleashed ERP. The challenge wasn't just migrating data; it was rebuilding their entire operational workflow around a fundamentally different system architecture, without disrupting live business operations.

What We Built

Led a complete data and workflow migration from MYOB to Unleashed ERP, reverse-engineering a 150-table legacy schema including XML data from handheld PDA hardware.
Rebuilt core production and operational workflows from scratch to align with modern ERP architecture standards rather than just porting legacy behavior.
Designed backend services for inventory, order, and product lifecycle management within the new Unleashed environment.
Integrated external data sources and ensured seamless cross-system synchronization throughout the transition window.
Executed a staged migration strategy that guaranteed zero data loss and business continuity throughout the cutover.
Outcome

The client transitioned to a modern ERP system with improved operational efficiency, streamlined workflows, and significantly better inventory visibility — with no data loss and no downtime.

Project 06

Agentic RAG
Sales System

Type
AI / Agentic System
Status
R&D / Internal
Tech Stack
LangGraph PGVector OpenAI FastAPI PostgreSQL

The Problem

Standard RAG chatbots retrieve documents and answer questions. That's useful, but it's not an agent — it can't reason across linked content, explore context chains, or guide a user toward a sale. We wanted to build something that could hold a meaningful product conversation grounded in real catalog data.

What We Built

Built an agentic RAG application using LangGraph — the agent reasons over retrieved content rather than just surfacing it.
Implemented semantic search with PGVector for context-aware retrieval across product and content pages, ranking results by relevance to the user's actual intent.
Added linked-context expansion — the agent follows semantic relationships between documents, pulling in related content the user didn't explicitly ask for but needs.
Designed a multi-step reasoning loop that lets the agent plan retrieval, evaluate what it found, and decide whether to expand context before responding.
Built the system as a deployable FastAPI service, ready to integrate into any product or content-heavy sales surface.
Outcome

A working agentic RAG system that goes beyond document retrieval into genuine reasoning over a product catalog — the foundation for production sales agents we're building on top of.
